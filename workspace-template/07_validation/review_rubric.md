# Review Rubric

Purpose: Define how outputs should be evaluated before they are accepted.

## Scoring Dimensions
| Dimension | What Good Looks Like |
|---|---|
| Relevance | Directly addresses the task |
| Specificity | Includes concrete steps, examples, or decisions |
| Actionability | Can be used immediately |
| Evidence | Clearly states assumptions and support |
| Risk awareness | Flags important risks and mitigations |

## Acceptance Rule
- 24-30: Accept
- 18-23: Accept with minor revisions
- 12-17: Revision required
- Below 12: Re-scope the task
