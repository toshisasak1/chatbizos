# LLM プロバイダー

## OpenAI の設定

### API Key

- OpenAI ダッシュボードで API キーを作成します。
- `.env` の `OPENAI_API_KEY` に設定します。
- `LLM_PROVIDER=openai` を指定します。

### OAuth

OpenClaw の構成によってはアカウントベースの認証フローに対応できる場合があります。ただし、このリポジトリの標準セットアップは、セルフホスト前提でより単純な API キー方式を前提にしています。

### コストの考え方

OpenAI のコストは、使うモデルと利用量に依存します。調査タスクが多い場合や、長いマルチエージェントセッションを頻繁に回す場合はコストが上がります。

## Anthropic の設定

- Anthropic の API キーを作成します。
- `.env` の `ANTHROPIC_API_KEY` に設定します。
- `LLM_PROVIDER=anthropic` を指定します。

Anthropic を他プロジェクトでも使っている場合や、モデル挙動の好みが合う場合に向いています。

## ローカル LLM の設定

- Ollama などのローカルモデルサーバーを起動します。
- たとえば `http://localhost:11434` に到達できることを確認します。
- `LLM_PROVIDER=local` を指定します。
- `LOCAL_LLM_ENDPOINT` に正しい URL を設定します。

ローカルモデルは品質や応答安定性に差が出やすい点に注意してください。

## プロバイダーを切り替える

切り替え手順:

1. `.env` の `LLM_PROVIDER` を変更する
2. 対応する認証情報が入っていることを確認する
3. コンテナを再起動する

## プロバイダー別のコスト感

- **OpenAI**: 品質が安定しやすいが、API コストは利用量に応じて変動
- **Anthropic**: 利用パターンは近いが、モデルとプロンプト長でコストが変わる
- **Local**: 初期準備後の追加コストは低いが、品質と運用負荷のトレードオフがある

初めて使う場合は、すでに運用に慣れているプロバイダーから始めるのが無難です。
