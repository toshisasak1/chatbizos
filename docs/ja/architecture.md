## エージェント間の通信フロー

ChatBizOS はスター型トポロジーを使います。中心にいるのは Producer です。

1. 人間が Discord または Slack からメッセージを送ります。
2. Producer がワークスペースの文脈を踏まえて依頼を解釈します。
3. Producer が `STATUS.md` と `task_queue.md` を読みます。
4. 適切な専門エージェントに `[TASK]` メッセージを送ります。
5. 専門エージェントが担当ファイルを更新します。
6. Producer が出力を評価し、受理するか修正依頼を返します。
7. 重要な判断やリスクは人間に戻して確認します。

## タスクループ

デフォルトのループは次のとおりです。

- 状態を読む
- 次の優先課題を決める
- 担当者に依頼する
- 結果をレビューする
- 判断を記録する
- 次のアクションをキューに入れる

この流れによって、チームがバラバラなチャットボットとして動くのではなく、1 つのワークフローとして進みます。

## メモリアーキテクチャ

ChatBizOS は 3 層の記憶を使います。

1. **セッションメモリ**: 直近の会話文脈
2. **構造化された学習ログ**: `LEARNINGS.md`
3. **ワークスペース文書**: 実際の状態を残す Markdown ファイル群

最も重要なのはワークスペース文書です。セッションを跨いでも残り、人間がいつでも確認できます。

## ワークスペース構成

- `01_strategy/`: 事業の方向性と顧客定義
- `02_research/`: 市場情報と競合観察
- `03_offer/`: オファーと価格設計
- `04_growth/`: マーケティング施策とコンテンツ
- `05_implementation/`: ツール選定と実装手順
- `06_operations/`: キュー、ログ、日次サマリー
- `07_validation/`: 実験、指標、評価基準、コンプライアンス確認
