# トラブルシューティング

## Gateway が起動しない

次を確認してください。

- Docker のビルドログ
- `18789` ポートがすでに使われていないか
- 生成された設定ファイルが正しいか
- イメージ内で `openclaw` が正しくインストールされているか

## Discord Bot が反応しない

次を確認してください。

- `DISCORD_BOT_TOKEN`
- `DISCORD_GUILD_ID`
- `DISCORD_CHANNEL_ID`
- サーバーとチャンネルでの Bot 権限
- `CHAT_CHANNEL=discord` になっているか

## Slack Bot が反応しない

次を確認してください。

- `SLACK_BOT_TOKEN`
- `SLACK_APP_TOKEN`
- `SLACK_CHANNEL_ID`
- App がワークスペースにインストールされているか
- `CHAT_CHANNEL=slack` になっているか

## LLM API エラーが出る

次を確認してください。

- 正しいプロバイダーが選ばれているか
- 対応する API キーが設定されているか
- キーがまだ有効か
- 外向き API 通信がネットワーク上で許可されているか

## エージェントが出力しない

次を確認してください。

- ワークスペースファイルが存在しているか
- 対応する `SOUL.md` が配置されているか
- Producer が明確なタスクを委譲しているか
- 人間確認が必要な状態で停止していないか

## ワークスペースファイルが更新されない

次を確認してください。

- `./workspace` がコンテナにマウントされているか
- ホスト側のファイル権限
- コンテナが想定どおりの `WORKSPACE_DIR` で起動しているか
- エージェントが別のワークスペースパスへ書き込んでいないか
