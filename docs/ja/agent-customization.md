# エージェントのカスタマイズ

## SOUL.md はどう効くか

各エージェントの性格と振る舞いは、主に `SOUL.md` によって決まります。`SOUL.md` には、そのエージェントの役割、優先順位、品質基準、エスカレーションルール、作業スタイルが定義されています。

つまり `SOUL.md` を変えると、そのエージェントの行動が変わります。

## エージェントの振る舞いを調整する

エージェントを変更するときは、次を意識してください。

1. まずは最小限の変更で目的を達成する
2. 役割の境界を曖昧にしない
3. 例、品質ルール、エスカレーション条件をまとめて更新する
4. 特定の顧客情報や環境依存の前提を埋め込まない

## 新しいエージェントを追加する

新しいエージェントを追加する手順:

1. `agents/` 配下に新しいディレクトリを作る
2. `SOUL.md` を追加する
3. `config/openclaw-config.json` の生成ロジックを更新する
4. Producer のタスクループにどう組み込むか決める
5. ドキュメントと examples を更新する

## エージェントを減らす

エージェントを削除するときは、次を行います。

1. `agents/` 配下の対象ディレクトリを削除またはアーカイブする
2. 生成される OpenClaw 設定から外す
3. タスクキュー運用ルールの記述を見直す
4. 他エージェントの担当範囲を広げる必要があるか確認する

## ドメイン特化の構成を作る

よいドメイン特化カスタマイズは、次の条件を満たします。

- 基本の運用パターンは維持する
- 業界固有のチェックリストを追加する
- 例や判断ヒューリスティクスを差し替える
- 目的が明確でない限り、狭すぎる単一ケースに過剰適合させない

例:

- EC
- SaaS
- コンサルティング
- 教育商品
- 地域サービス業
